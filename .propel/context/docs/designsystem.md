# Design Tokens

## Colors
- Primary: #2563eb (blue)
- Secondary: #64748b (slate)
- Accent: #22d3ee (cyan)
- Success: #16a34a (green)
- Warning: #f59e42 (orange)
- Error: #dc2626 (red)
- Background: #f8fafc (light)
- Surface: #ffffff (white)
- Text: #0f172a (dark)

## Typography
- Font Family: Inter, Arial, sans-serif
- Headings: 700/600 weight
- Body: 400 weight
- Caption: 300 weight
- Sizes: 32, 24, 20, 16, 14, 12 px

## Spacing
- XS: 4px
- SM: 8px
- MD: 16px
- LG: 24px
- XL: 32px

## Border Radius
- Small: 4px
- Medium: 8px
- Large: 16px

## Shadows
- Card: 0 2px 8px rgba(0,0,0,0.06)
- Modal: 0 4px 16px rgba(0,0,0,0.12)

## Iconography
- Feather Icons, 24px grid

# Component Specifications
- Button: Primary, Secondary, Icon, Disabled
- Input: Text, Password, Email, Date, Select
- Card: Patient, Appointment, Notification
- Table: Appointments, Users, Audit Logs
- Modal: Confirmation, Error, Form
- Tab: Dashboard, Management, Profile
- Alert: Success, Warning, Error
- Avatar: User, Patient
- Badge: Status, Role

# Branding
- Logo: Simple, healthcare-themed, blue/cyan palette
- Favicon: Platform logo, 32x32px

# Accessibility
- All components must meet WCAG 2.1 AA
- Sufficient color contrast
- Keyboard navigation support
- ARIA labels for all interactive elements

# Responsive Guidelines
- Desktop: 1440px
- Tablet: 768px
- Mobile: 375px
- Fluid layouts, mobile-first breakpoints

# Notes
- All tokens and components are extensible for future needs.
