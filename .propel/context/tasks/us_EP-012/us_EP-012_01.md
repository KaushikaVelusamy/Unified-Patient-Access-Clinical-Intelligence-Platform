# User Story: Role-Based Access & Audit Logging

## Epic: EP-012

### As an admin, staff, or patient,
I want the system to enforce strict role-based access control and log all actions immutably,
so that compliance and security are ensured for all users.

#### Acceptance Criteria
- The system checks user roles and permissions for every action.
- Access is granted or denied based on role.
- All actions (granted or denied) are logged immutably for audit.
- Audit logs are accessible to authorized admins.
- System alerts admin if audit log storage fails.

#### Business Value
Ensures compliance, enhances security.

#### Effort Estimation
8 Story Points
